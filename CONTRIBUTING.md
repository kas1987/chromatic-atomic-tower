# Contributing to CAT

## Before changing files

1. Identify Mission ID.
2. Identify BEAD ID.
3. Confirm the target file is in `allowed_paths`.
4. Run validation after changes.
5. Attach evidence or explain why no evidence applies.

## Commit format

```text
[MISSION_ID][BEAD_ID] Imperative summary
```

Example:

```text
[MP-CAT-000][BEAD-CAT-000-002] Add mission schema validation
```

## Pull request rules

Each PR must include:

- Mission ID
- BEAD ID
- objective
- files changed
- validation run
- evidence path
- risks
- reviewer checklist
