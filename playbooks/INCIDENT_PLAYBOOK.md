# Incident Playbook

## Trigger an incident when

- forbidden path is touched
- tool budget is exceeded without approval
- validation is bypassed
- secret or credential appears
- mission or BEAD is mutated without traceability
- agent enters a loop
- data loss or destructive action occurs

## Immediate response

1. Stop work.
2. Record incident.
3. Preserve evidence.
4. Contain damage.
5. Request human decision.
6. Add learning after resolution.
