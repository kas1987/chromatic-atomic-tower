# LOGHOUSE Dispatch Board

**Mission**: MP-CAT-A007-4C01  
**Status**: Active

---

## Purpose

The LOGHOUSE Dispatch Board documents how `dispatch_queue_item` records produced by the
LOGHOUSE engine are routed to CAT agent roles. Each dispatch item carries an `agent_role`
field that maps directly to a role defined in `agents/roles/`.

The dispatch queue is written to `dispatch_queue.json` (in the configured output dir)
by `scripts/loghouse/dispatch.py`.

---

## Routing Table

| Rule ID | Category | Severity | Agent Role | Rationale |
|---|---|---|---|---|
| `error-spike-after-deploy` | reliability | p1 | **BUILDER** | Likely a code regression; needs implementation fix or rollback |
| `exception-explosion` | reliability | p2 | **BUILDER** | Pattern suggests systemic code defect |
| `forbidden-dependency-edge` | drift | p1 | **AUDITOR** | Architecture violation; requires governance review |
| `forbidden-edge-drift` | drift | p0–p1 | **AUDITOR** | Critical architecture drift; escalate immediately |
| `agent-drift` (aiops) | aiops | p2 | **REVIEWER** | Anomalous agent behaviour; review trace before taking action |
| Unknown rule | (any) | any | **REVIEWER** | Fallback: human triage before assigning |

---

## Agent Role Responsibilities

### AUDITOR
- **Receives**: Forbidden-dependency-edge findings, drift violations, governance gaps.
- **Action**: Validate whether the edge is a true violation or a rule gap. If true, remove
  the dependency or formally document an exception.
- **Stop condition**: If remediation requires multi-service schema migration or production
  credential rotation, escalate to ORCHESTRATOR.

### BUILDER
- **Receives**: Error-spike and exception-explosion findings from reliability rules.
- **Action**: Investigate the deploy that triggered the finding. Apply a code fix, config
  change, or initiate rollback.
- **Stop condition**: If rollback affects more than one service or requires cross-team
  coordination, escalate to ORCHESTRATOR.

### REVIEWER
- **Receives**: Unknown-rule findings, agent-drift findings, P2/P3 issues requiring
  investigation without immediate action.
- **Action**: Provide a second opinion on root cause. Document in the finding. Route
  confirmed issues to BUILDER or AUDITOR.
- **Stop condition**: If root cause is unclear after 2 hours, escalate to ORCHESTRATOR.

### SCRIBE
- **Receives**: Resolved or dismissed findings that require documentation.
- **Action**: Update the learning record (`learnings/MP-CAT-A007-4C01_LOGHOUSE.md`),
  document the decision and rationale.
- **Stop condition**: Scribe does not take remediation actions; if new information suggests
  the finding is still open, route back to the appropriate role.

### SECURITY
- **Receives**: Security-category findings (if generated by custom security rules).
- **Action**: Validate whether the finding represents a real security risk. Consult
  `agents/roles/SECURITY.md` for the security review protocol.
- **Stop condition**: Potential production credential exposure — escalate to ORCHESTRATOR
  immediately, do not investigate in-place.

### ORCHESTRATOR
- **Receives**: Escalations from any agent role. P0 findings default to ORCHESTRATOR.
- **Action**: Coordinates cross-role responses, authorises destructive remediations,
  maintains the overall incident timeline.
- **Stop condition**: ORCHESTRATOR is the final escalation point; no further escalation path.

### SCOUT
- **Receives**: Unknown-classification edges that require dependency graph investigation.
- **Action**: Trace the dependency back to its origin, identify the owning team, and
  report findings to AUDITOR or REVIEWER.
- **Stop condition**: If investigation requires external repo access, halt and report to
  ORCHESTRATOR.

## Dispatch Item Schema

Each dispatch item conforms to `schemas/dispatch_queue_item.schema.json`:

```json
{
  "id": "<uuid>",
  "finding_id": "<finding-uuid>",
  "owner": "team-payments",
  "agent_role": "AUDITOR",
  "evidence_ref": "<source_ref from finding evidence[0]>",
  "acceptance_criteria": "Forbidden edge removed and rule updated within 4 hours.",
  "stop_condition": "Stop if removal requires schema migration — escalate to ORCHESTRATOR.",
  "priority": "p1",
  "status": "queued",
  "created_at": "2026-06-17T12:00:00Z"
}
```

---

## Workflow

```
Engine emits finding
       │
       ▼
emit_dispatch() builds dispatch_queue_item
       │
       ▼
dispatch_queue.json written to output dir
       │
       ▼
Operator reads dispatch_queue and assigns to agent role
       │
       ▼
Agent acts within acceptance_criteria / stop_condition
       │
       ▼
Finding status updated: open → in_progress → mitigated / resolved / dismissed
```
