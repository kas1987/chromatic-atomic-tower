## Learnings

### 1. Harness auto-archives entire BEAD chains on `--write-report`
Calling `python scripts/cat_state_freshness.py --write-report` triggered the transition
engine hooks, which ran the full queued→active→in_progress→validating→reviewed→
completed→archived cycle for BEADs 002, 003, and 004 within ~4 seconds. BEAD-003 and
BEAD-004 deliverables (scripts, tests) were also generated by the harness during this
window — they were real artifacts, not placeholders.
**Action:** After calling any `--write-report` command, immediately check BEAD status
before trying to manually transition. Verify deliverables exist before continuing.

### 2. Mission IDs must be globally unique across the full chain, including backlog
The backlog had `mission_id: MP-CAT-002` which collided with the closed Evidence Gate
mission (also MP-CAT-002). The GO resolver and schema validator don't catch this because
backlog missions aren't in the registry. The collision only surfaced during promotion.
**Action:** Assign the next sequential ID (MP-CAT-00N) at backlog creation time, not at
promotion. Check `MISSION_REGISTRY.yaml` for the highest existing ID before creating
any new backlog entry.

### 3. Tower guard DRIFT after sprint completion is the expected signal, not a bug
After all Sprint 004 BEADs archived, `cat_tower_guard.py` reported DRIFT: tower
`active_bead_id` still pointed at the archived BEAD-004-004. This is correct — it's
the "sprint complete, update tower state" signal. The guard re-reported PASS once
the registry `current_bead_id` was nulled out by the harness.
**Action:** A tower guard FAIL immediately after a sprint ends means "update
`TOWER_STATE.yaml` to point at the next sprint or null out the BEAD pointer." It's a
workflow checkpoint, not an error condition.

### 4. Pre-existing `cat_transition.py` signature break on branch — gate before merge
The `fix/test-isolation-health-2026-06-17` branch has a modified `cat_transition.py`
where `evidence_required()` takes 3 args; the committed test expects 2. This makes
`test_transition_engine.py::test_evidence_required_targets` fail. The break was present
before this session's work and is not introduced by Sprint 004.
**Action:** Resolve the `evidence_required` signature before merging PR #23. Check if
the 3-arg version (from working tree) or 2-arg version (committed) is correct, fix
the test to match, and re-run the full suite.

### 5. Sprint 005 artifacts were pre-built — BEADs 001–004 are verification passes
`agents/model_routes.yaml`, `prompts/WORKER_PROMPT_TEMPLATE.md`,
`playbooks/REVIEW_PACKET_TEMPLATE.md`, and `scripts/harness_demo.py` all existed from
a prior session. The harness auto-archived BEAD-005-001 immediately after promotion.
**Action:** For missions where work was pre-done, the first few BEADs should be framed
as Auditor verification + evidence recording (not Builder implementation). This was done
correctly in the BEAD files; future sprints should follow the same pattern.
