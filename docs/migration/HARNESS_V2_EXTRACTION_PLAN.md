# Harness V2 Extraction Plan

Harness V2 is donor material, not the CAT foundation.

## Extract

- M1-M4 mission complexity model
- Mission Packet structure
- BEAD concept
- GO-mode ideas
- Orchestrator, Builder, Reviewer, Auditor, Scribe roles
- Confidence gate
- Tool budgets
- Review and closeout checklists
- ChromaticTrees repo organization rules
- Learning and incident loops

## Do not directly migrate

- duplicated playbooks
- outdated experiments
- vague agent instructions
- broad docs without operational authority
- half-built scaffolds
- old naming that conflicts with CAT

## Extraction rule

Each migrated idea needs one of:

- mission update
- BEAD update
- playbook update
- schema update
- decision log entry

No direct bulk copy.
