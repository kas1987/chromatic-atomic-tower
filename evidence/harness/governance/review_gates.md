# Review Gates

## Gate 1: Ticket Completeness

A ticket is dispatchable only if it has:

- objective
- context
- allowed files
- blocked actions
- required commands
- evidence requirements
- acceptance criteria
- stop conditions

## Gate 2: Worker Output Completeness

Worker output is reviewable only if it has:

- summary
- changed files
- commands run
- test results
- known risks
- escalation notes

## Gate 3: Evidence Packet Completeness

Final review requires:

- ticket
- diff or patch
- test/lint output
- cheap review notes
- worker notes
- known failures

## Gate 4: Merge Readiness

Merge requires:

- passing required validation
- no blocking findings
- Opus approval
- human approval
